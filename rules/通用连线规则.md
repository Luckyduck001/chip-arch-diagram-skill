# 芯片架构图 — 通用连线规则

> 执行顺序：Phase 1（内部↔外围）→ Phase 2（直连）→ Phase 3（L 形）。不可变更。

---

## 一、通用规则

### 1.1 正交

只允许水平和垂直线段。禁止斜线。

### 1.2 不穿框

连线不得穿过任何非源/目标模块的内部。

### 1.3 箭头方向

每条连线生成后，验证 SVG 终点落在数据流**目标模块**的边上。若落在源边 → 翻转坐标顺序。

### 1.4 连线样式（可配置）

默认黑色实线 `stroke="black" stroke-width="1.5" fill="none"`。箭头 marker 按连线颜色分别生成（`ar_black`、`ar_red` 等），确保箭头颜色与线一致，引用 `marker-end="url(#ar_black)"`。

用户可通过行尾 `[...]` 自定义：
- `颜色`/`color`：线颜色（如 `red`、`#FF0000`）
- `线形`/`style`：`实线`/`solid`、`虚线`/`dashed`（stroke-dasharray="6,4"）、`点线`/`dotted`（stroke-dasharray="3,3"）
- `描述`/`label`：文字描述，渲染在连线中点上方偏移 8px，带白色半透明背景

### 1.5 连线标签渲染

- Phase 1/2 直线：中点沿垂直方向偏移 8px（水平线→上方，垂直线→左侧）
- Phase 3 L 形：首段中点，偏移规则同上
- 标签字号 7px，颜色 #555，白色背景 `fill="white" opacity="0.85"` 防遮挡
- 仅当 `line_label` 非空时渲染

### 1.6 不穿框检测

对所有多单元格模块（col_span>1 或 row_span>1），连线不得穿过其内部。线段与任何非源/目标模块的矩形区域重叠时报警告。

---

## 二、Phase 1 — 内部↔外围

- 连线为 **1 段直线**
- **内部优先**：内部模块边定等分点（N=K+1, K=该边连接数），外围模块边投影对齐
- 方向：按数据流确定起止点（源→目标），不可固定方向

### 处理流程

| 步骤 | 操作 |
|------|------|
| 1. 算边 | 外围在 c=0 → 内部用左边；外围在 c=max → 内部用右边。汇总每条内部边的连接数 K |
| 2. 定点 | 内部边等分：出入点 = 边起点 + k × 边长 / N。多条时按投影点坐标排序 |
| 3. 投影 | 外围端点 = 内部端点投影到外围边上（水平线同 y，垂直线同 x） |
| 4. 画线 | 1 段直线，箭头在终点 |

---

## 三、Phase 2 — 内部↔内部直连

- 行有重叠 → 水平直线，对齐到重叠区域像素中心
- 列有重叠 → 垂直直线，对齐到重叠区域像素中心
- 出入点使用预计算的 k 和 N：出入点 = 边起点 + k × 边长 / N
- 若该边无其他连接共享，C=1，N=2，k=1 → 中点

---

## 四、Phase 3 — 内部↔内部 L 形 / Z 形

> 用户必须用 `【L】A（边）→ B（边）` 指定出入边，不写报错。同向边（如 右→右、底→底）自动画 Z 形（3段），异向边（如 右→底、左→顶）画 L 形（2段）。

**路由处理顺序**：按源模块排序——r 小→大，r 相同则 c 小→大。先处理的先占轨道。

### 4.1 边连接数汇总

Phase 1 + Phase 2 + Phase 3 连接统一统计（在 SKILL.md 步骤 4.0），逐模块逐边：

```
模块 X 底边: [Phase2 X→Y (k=1)] [Phase3 Z→X] → C=2, N=3
模块 X 右边: [Phase1 PIO→X] → C=1, N=2
```

### 4.2 k 值分配

对同一条边的所有连接：
1. **Phase 3 L/Z 形优先**：遍历 k=1..N-1，每个 k 计算到对端边 k=1 位置的曼哈顿距离，选最短的 k
2. **Phase 2 填剩余**：按到对端模块中心的距离排序，从小到大填剩余 k
3. N = C + 1

### 4.3 统一 N + 出入点计算

每条连接涉及源边（N₁）和目标边（N₂）：
- N_unified = max(N₁, N₂)
- 源出点 = 源边起点 + k × 源边长 / N_unified
- 目标入点 = 目标边起点 + k × 目标边长 / N_unified

### 4.4 L 形 / Z 形路由

**异向边（L 形，2 段）**：源边和目标边不同方向（如右→底、左→顶）。

```
若源边为左/右（水平边）→ 首段水平，拐角 = (目标入点.x, 源出点.y)
若源边为顶/底（垂直边）→ 首段垂直，拐角 = (源出点.x, 目标入点.y)
路径 = [源出点, 拐角, 目标入点]，严格 2 段。
```

**同向边（Z 形，3 段）**：源边和目标边同方向（如右→右、底→底）。

```
若同为左/右 → 水平→垂直→水平
  mid_x = (源出点.x + 目标入点.x) / 2
  路径 = [源出点, (mid_x, 源出点.y), (mid_x, 目标入点.y), 目标入点]

若同为顶/底 → 垂直→水平→垂直
  mid_y = (源出点.y + 目标入点.y) / 2
  路径 = [源出点, (源出点.x, mid_y), (目标入点.x, mid_y), 目标入点]
```

- 首段 ⟂ 源边
- 末段 ⟂ 目标边

### 4.5 交叉检测

逐条路由后，检测新路径各段与已路由段是否正交交叉。

**公式（3.11）**：水平段 H(y_h, x₁~x₂) 与垂直段 V(x_v, y₁~y₂) 相交 ⇔

```
x_v ∈ [min(x₁,x₂), max(x₁,x₂)] ∧ y_h ∈ [min(y₁,y₂), max(y₁,y₂)]
```

**排除共享端点**：若交点恰为 H 或 V 的端点（坐标差 < 1px），不算交叉。

**冲突处理**：
1. 该边对有多个可用 k → 换 k 重试
2. 该边对仅一个 k（C=1）→ 报告用户，建议调整出入边
3. 全部 k 均交叉 → 报告用户

## 五、关键公式速查

### 出入点

```
出入点 = 边起点 + k × 边长 / N
N = max(C₁, C₂) + 1（统一 N）
```

### 交叉检测

```python
def crosses(h_seg, v_seg):
    hx1, hx2 = sorted([h_seg[0].x, h_seg[1].x])
    vy1, vy2 = sorted([v_seg[0].y, v_seg[1].y])
    vx, hy = v_seg[0].x, h_seg[0].y
    if not (hx1 <= vx <= hx2 and vy1 <= hy <= vy2):
        return False
    # Exclude shared endpoints (within 1px tolerance)
    if (abs(vx - h_seg[0].x) < 1 or abs(vx - h_seg[1].x) < 1 or
        abs(hy - v_seg[0].y) < 1 or abs(hy - v_seg[1].y) < 1):
        return False  # shared endpoint, not a cross
    return True
```
