# 芯片架构图 — 通用布局规则

> 本文件定义**静态布局**——网格、模块、父框、颜色、对齐。
> 连线路由规则见 `通用连线规则.md`。

---

## 一、网格坐标

所有模块放在 (c, r) 网格中，每个模块占 w×h 格。

- c = 列号 (0, 1, 2, ...)，间距统一
- r = 行号 (0, 1, 2, ...)，行高统一
- w = 列跨度，h = 行跨度

除外围模块外，所有内部模块 w=1, h=1。

---

## 二、模块分类与尺寸

| 类型 | w×h | 特征 |
|------|-----|------|
| 内部模块 | M×N | 默认1×1，可在 Excel 中跨多格 |
| 外围模块 | 1×N 或 M×1 | 竖长条/横条，位于四边 |
| 父框 | M×N | 包裹子模块的外接矩形 |

---

## 三、外围模块

外围模块位于四角，竖长条 (h > 1)：

- **c=0 列**：左侧外围，从上到下排列
- **c=max 列**：右侧外围，从上到下排列

高度 = 覆盖行数 × ($CELL_H + $ROW_GAP) − $ROW_GAP。确保上下相邻的外围模块不重叠。

---

## 四、行分配

1. **主数据流独占一行**。每条外部→外部的完整数据路径放在同一行，模块按数据流顺序从左到右排列，全水平连线。
2. **不同主数据流不同行**。避免水平连线重叠。
3. **分支模块放异行**。从主数据流分出的模块放在其他行，通过垂直连线接入。
4. **同一列的模块形成垂直链**，便于数据上下传递和反馈。

---

## 五、父框

父框分两级：

**内框**：包裹一个子系统的子模块（如 GRP_A、GRP_B、GRP_C）。

**外框**：包裹所有内部模块（不含外围模块），取内框（父框）的最小外接矩形，各边向外额外扩展一层边距，确保外框与内框不重叠。

### 5.1 父框坐标格式（强制）

> ⚠️ **历史 Bug**：AI 曾把 `{c范围, r范围}` 错误解析为 `{r, c}`，导致所有父框包裹了错误的模块。

用户输入格式：`父框名 {c范围, r范围}` — **先列后行**。c 和 r 都从 0 开始。

```
示例：
  CH_ALIGN {1-2, 0}     → c=1..2, r=0     （列1-2，第0行）
  PKT_TOP  {4-5, 0-2}   → c=4..5, r=0..2  （列4-5，第0-2行）
  RX_DSI_APB {1-2, 4}   → c=1..2, r=4     （列1-2，第4行）
```

**解析规则**：正则匹配 `{(\d+)-(\d+),\s*(\d+)(?:-(\d+))?}` 时，group 1-2 是列范围，group 3-4 是行范围。**禁止**假设 `{行, 列}` 的顺序。

**子模块匹配**：使用**区间重叠判断**而非包含判断：
```
module.r0 <= frame.r1  AND  module.r1 >= frame.r0  AND
module.c0 <= frame.c1  AND  module.c1 >= frame.c0
```
> 模块可能跨越多行（如外围模块 PCS 占 r=0-3），父框只覆盖其中一部分（r=1-2）。用包含判断会漏掉这种模块。

### 5.2 外框与外部模块

外部模块（灰色，c=0 或 c=max_col）应在**外框之外**。外框包裹内部模块 + 父框，与外部模块之间保留 ≥$COL_GAP 间距。

### 5.3 标签防遮挡（强制）

> ⚠️ **历史 Bug**：标签直接画在左上角被模块 rect 遮挡；加白色背景但 DOM 顺序错误导致模块盖住背景。

**标签位置**：父框标签在**左上角**（text-anchor="start"），外框标签在**左上角**。**不得**移到居中或其他位置。

**防遮挡方案**：每个标签前加一个白色半透明 `<rect>` 作为背景：
```xml
<rect x="lx-2" y="ly-font_size+1" width="text_width" height="font_size+2" 
      fill="white" opacity="0.9"/>
<text x="lx" y="ly" text-anchor="start" font-size=".." fill="#555">LABEL</text>
```

**DOM 绘制顺序（强制）**：
```
1. 外框 <rect>
2. 父框 <rect>（仅矩形，不含标签）
3. 模块 <rect> + 模块 <text>
4. 框标签背景 <rect> + 框标签 <text>   ← 必须在模块之后！
5. 连线 <line> + <polyline>
```
> 原因：SVG 使用画家算法（后绘制的在上层）。标签必须在模块之后才能在视觉上覆盖模块。步骤 4 的标签背景 rect 提供 90% 不透明度白色底色，阻挡下方模块内容。

**标签 y 坐标公式**（强制）：
```
label_y = max(框_y + font_size + 2, 子模块最小_y - 4)
```
> 确保标签基线至少有 4px 不侵入子模块区域。

### 5.4 框的顶部边距必须容纳标签（强制）

> ⚠️ **历史 Bug**：FRAME_PAD=12 放 8px 字号的标签，标签到模块只剩 1-2px，视觉上被遮挡。

父框顶边距应 ≥ 标签字号 + 6px。用户输入的"父框边距"值是**最小边距**，标签空间不足时应自动增大。

| 标签字号 | 最小顶边距 |
|---------|-----------|
| 8px | 14px |
| 9px | 16px |
| 10px | 18px |

外框顶边距应 ≥ 32px（硬编码地板，非公式推导，以容纳两层标签）。

## 六、框样式

### 6.1 父框和外框样式（可配置）

用户可在输入文件的「父框样式」段落中指定外框和各父框的 stroke、fill、style：

```
外框: stroke=#555, fill=none, style=solid
GRP_A: stroke=#555, fill=#F5F5F5, style=solid
```

默认值：`stroke=#555, fill=none, style=solid`（深灰色实线，无填充）。

### 6.2 Z-order（绘制顺序）

外框在最底层（最先绘制），父框在外框上一层，模块在父框之上：

```
① 背景 → ② 外框 → ③ 父框 → ④ 模块 → ⑤ 标签 → ⑥ 连线
```

父框有 fill 色时覆盖外框，但不会遮盖模块。

---

## 七、连线样式

默认黑色实线，无图例。用户可通过行内 `[...]` 属性自定义颜色、线形、文字描述：

```
MOD_A → MOD_B [颜色=红, 线形=虚线, 描述="数据总线"]
```

属性均为可选。线形支持：实线/solid, 虚线/dashed, 点线/dotted。文字标签渲染在连线中点上方（偏移 8px），带白色半透明背景。

## 八、颜色定义

### 7.1 颜色映射表（唯一真相源）

| 浅绿 | `#C5E0B4` | `#A9D18E` | 时钟域 |
| 深绿 | `#548235` | `#375623` | 电源域 |
| 浅紫 | `#D9C2EC` | `#B994D4` | — |

> ⚠️ 浅蓝的 fill 是 `#BDD7EE`（非 `#ADD8E6`）。所有颜色均配备 stroke，纯色/渐变均可使用。

### 7.2 渐变填充

跨时钟域模块使用左右双色渐变填充：左右两半各占约 48%，中间 4% 为快速过渡带，不采用缓慢渐变。

```
<linearGradient id="grad_模块名" x1="0" y1="0" x2="1" y2="0">
  <stop offset="0%"   stop-color="左色"/>
  <stop offset="48%"  stop-color="左色"/>
  <stop offset="52%"  stop-color="右色"/>
  <stop offset="100%" stop-color="右色"/>
</linearGradient>
```

模块边框色取填充色两端中更深的那一端。纯色填充则取比填充色更深一档的颜色（见上表 stroke 列）。

## 九、外围间距

外围模块与外框之间保留足够间距（≥ $COL_GAP），用于标注连线文字。

## 十、统一对齐

所有模块严格按网格坐标对齐：

- 同一行的模块 **y 坐标完全相同**
- 同一列的模块 **x 坐标完全相同**
- 所有内部模块 **w 完全相同**、**h 完全相同**
- 父框边距统一（上下左右等距），边距 = 定值
- 行高、列间距全局统一

### 参数定义

> 优先使用用户自定义值（来自输入文件 section 五），否则用默认值。所有坐标计算一律使用变量名，禁止硬编码数字。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `$LEFT_MARGIN` | 20 | 左侧外边距 |
| `$TOP_MARGIN` | 60 | 顶部外边距（含标签空间） |
| `$CELL_W` | 120 | 内部模块宽度 |
| `$CELL_H` | 60 | 内部模块高度 |
| `$PERIPH_W` | 60 | 外围模块宽度 |
| `$COL_GAP` | 50 | 列间距 |
| `$ROW_GAP` | 40 | 行间距 |
| `$FRAME_PAD` | 12 | 父框边距 |
| `$OUTER_PAD_TOP` | 20 | 外框顶边距 |
| `$OUTER_PAD_OTHER` | 8 | 外框其他边距 |

### 像素坐标公式

```
col_x[0] = $LEFT_MARGIN + $PERIPH_W

对 c > 0:
  若 c-1 列为内部列 → col_x[c] = col_x[c-1] + $CELL_W + $COL_GAP
  若 c-1 列为外围列 → col_x[c] = col_x[c-1] + $PERIPH_W + $COL_GAP

row_y[0] = $TOP_MARGIN
对 r > 0: row_y[r] = row_y[r-1] + $CELL_H + $ROW_GAP

内部模块:
  x = col_x[c0]
  y = row_y[r0]
  w = col_span × $CELL_W + (col_span − 1) × $COL_GAP
  h = row_span × $CELL_H + (row_span − 1) × $ROW_GAP

外围模块:
  x = col_x[c]  ← c=0 或 c=max
  y = row_y[r_start]
  w = $PERIPH_W
  h = 覆盖行数 × ($CELL_H + $ROW_GAP) - $ROW_GAP
```

不得手动偏移单个模块的 x、y、w、h。

### 9.1 模块标签垂直居中（强制公式）

> ⚠️ 标签 y 坐标必须用公式计算，禁止凭感觉填值。

模块中心 y：`center_y = module_y + h / 2`

SVG 文本基线偏移修正：文本视觉中心约在基线上方 `font_size × 0.35` 处。

**单行标签**：
```
text_y = center_y + font_size × 0.35
```
> 例：h=60, font_size=9 → text_y = module_y + 30 + 3 = module_y + 33

**双行标签**（名称 > 16 字符时使用）：
```
行间距 = font_size × 1.2（约 11px @ 9px字号）
line1_y = center_y - 行间距/2 + font_size × 0.35
line2_y = line1_y + 行间距
```
> 例：h=60, font_size=9, 行间距=11 → line1_y = module_y + 28, line2_y = module_y + 39

双行文本块整体视觉中心 = `(line1_y + line2_y) / 2 - font_size × 0.35`，代入公式验证应 ≈ center_y。

## 十一、推导流程

1. **读取 Excel 网格表**：每格填模块名，空为 `.`，跨行模块每行都填
2. **分类模块**：
   - c=0 或 c=max_col → 外围模块
   - 其余 → 内部模块（默认可跨多格，同一模块名在相邻格出现 → 自动合并）
3. **读取用户参数**：从输入文件取自定义尺寸，未填用默认值（§十参数表）
4. **计算像素坐标**：按 §十 公式——
   - col_x[c]、row_y[r]
   - 内部模块：x=col_x[c0], y=row_y[r0], w=col_span×CELL_W+(col_span−1)×COL_GAP, h=row_span×CELL_H+(row_span−1)×ROW_GAP
   - 外围模块：x=col_x[c0], y=row_y[r_start], w=$PERIPH_W, h=覆盖行数×($CELL_H+$ROW_GAP)−$ROW_GAP
5. **计算父框**：子模块外接矩形 + $FRAME_PAD（四边统一）
6. **计算外框**：内部模块 ∪ 所有父框的合并外接矩形 + 外边距（顶=$OUTER_PAD_TOP，其他=$OUTER_PAD_OTHER）。无外框名时跳过标签
7. **应用颜色**：查 §八 颜色映射表，纯色取 fill+stroke，渐变用 §7.2 快切格式
8. **画模块**：严格按坐标，绘制顺序 = 外框 → 父框 → 模块 rect → 框标签 → 模块标签
