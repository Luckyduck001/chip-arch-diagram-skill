# 芯片架构图 — 通用检查清单

> 方法论：**约束验证**——从 HTML 提取值，验证约束条件是否满足。不重算期望坐标。

---

## A. 布局检查

- [ ] A1: 所有内部模块是否在外框内
- [ ] A2: 父框是否包裹其全部子模块（含边距 $FRAME_PAD）
- [ ] A3: 外框标签是否在框内左上角，不被模块遮挡（无外框名则 N/A）
- [ ] A4: 父框标签是否不被子模块遮挡（无标签则 N/A）
- [ ] A5: 外框顶边距 ≥ $OUTER_PAD_TOP

## B. 模块检查

- [ ] B1: 模块颜色（fill/stroke）是否与用户输入一致
- [ ] B2: 跨时钟域渐变是否为快切：0%→48% 左色，48%→52% 过渡，52%→100% 右色
- [ ] B3: 纯色模块边框是否比填充色更深一档
- [ ] B4: 模块标签是否垂直居中，不超出模块宽度
- [ ] B5: 多单元格模块宽高 = col_span×CELL_W+(col_span-1)×COL_GAP, row_span×CELL_H+(row_span-1)×ROW_GAP

## C. 连线通用检查

- [ ] C1: 每段是否纯水平/垂直（禁止 dx≠0 且 dy≠0）
- [ ] C2: 线段是否穿过非源/目标模块内部
- [ ] C3: 每条连线是否带 marker-end 箭头
- [ ] C4: DOM 顺序：模块在前，连线在后
- [ ] C5: 箭头方向：SVG 终点是否在数据流目标模块边上
- [ ] C6: 连线数量：HTML 中 `<line>` + `<polyline>` 总数 = 预期数
- [ ] C7: 连线颜色与用户指定的 `颜色`/`color` 属性一致（默认 black）
- [ ] C8: 连线线形与用户指定的 `线形`/`style` 属性一致（dashed→stroke-dasharray="6,4"）
- [ ] C9: 连线标签（`描述`/`label`）渲染在正确位置并有白色背景 rect

## D. Phase 1 — 内部↔外围

- [ ] D1: 连线是否为 1 段直线（`<line>`，非 `<polyline>`）
- [ ] D2: 内部模块端点是否在边等分点上（±1px）
- [ ] D3: 外围模块端点是否与内部端点投影对齐（水平线同 y，垂直线同 x）

## E. Phase 2 — 直连

- [ ] E1: 出入点 = 边起点 + k × 边长 / N，k=1
- [ ] E2: 直连是否未被 Phase 3 L 形挤占（k=1 永远是直连）

## F. Phase 3 — L 形 / Z 形

- [ ] F1: 异向边 L 形 = 2 段，同向边 Z 形 = 3 段
- [ ] F2: 首段 ⟂ 源边（源边垂直 → 首段水平；源边水平 → 首段垂直）
- [ ] F3: 末段 ⟂ 目标边
- [ ] F4: 出入点不在模块四角（k ≠ 0 且 k ≠ N）
- [ ] F5: k 值按「更近原则」分配（距离近的拿小 k）
- [ ] F6: Phase 2 直连固定 k=1，Phase 3 从 2 起（有直连时）
- [ ] F7: 交叉检测：任意两条路径的线段是否正交交叉

## G. 坐标格式

- [ ] G1: 父框 `{c范围, r范围}` 是否按**先列后行**解析（禁止假设 {r, c} 顺序）
- [ ] G2: 父框子模块匹配是否用**重叠判断**（`mod.r0 <= frame.r1 AND mod.r1 >= frame.r0`），非包含判断
- [ ] G3: 名称映射是否覆盖 Excel 缩写→MD 全称的全部模块（有未映射模块 → 报错）

## H. 标签遮挡

- [ ] H1: 标签 `<text>` 是否在模块 `<rect>` **之后**绘制（DOM 顺序）
- [ ] H2: 每个框标签前是否有白色半透明背景 `<rect>`（`fill="white" opacity="0.9"`）
- [ ] H3: 父框顶部边距 >= 标签字号 + 6px（如不够 → 自动增大 FRAME_PAD，不改标签位置）
- [ ] H4: 外框顶部边距 >= 外框标签字号 + 父框标签字号 + 8px
- [ ] H5: 标签在左上角（text-anchor="start"），未被移动到居中或其他位置
- [ ] H6: 标签背景 `<rect>` 位置和尺寸是否覆盖标签文字（y=ly-font_size+1, h=font_size+2）

## J. 框样式

- [ ] J1: 外框 stroke/fill/style 是否与「父框样式」段落一致（默认 #555/none/solid）
- [ ] J2: 父框 stroke/fill/style 是否与「父框样式」段落一致
- [ ] J3: 父框 rect DOM 位置是否在外框之后、模块之前（Z-order 正确）
- [ ] J4: 框有 fill 色时是否遮盖下层框但不遮盖模块

## K. 连线样式与标签

- [ ] K1: 连线颜色是否与用户指定 `颜色`/`color` 属性一致
- [ ] K2: 连线线形是否与用户指定 `线形`/`style` 属性一致（dashed→"6,4", dotted→"3,3"）
- [ ] K3: 连线标签（`描述`/`label`）位置 = 线段中点偏移 8px
- [ ] K4: 连线标签有白色半透明背景 rect 防遮挡

## I. SVG DOM 顺序

- [ ] I1: 绘制顺序 = 外框 rect → 父框 rect → 模块 rect+text → 框标签 bg+text → 连线
- [ ] I2: 连线数量 = Phase1数 + Phase2数 + Phase3数（`<line>` + `<polyline>` 总和）

---

## 违规记录格式

```
✗ 编号: 违反的约束 → HTML 实际值 → 修复方案

示例：
✗ F3: MOD_K→MOD_I 末段不⟂目标边（末段水平，目标边=左/垂直边）
  → 修复：检查 L 形拐角计算逻辑
✗ F7: P3_A 水平段(y=160) 与 P3_B 垂直段(x=780) 正交交叉
  → 修复：换 k 值或报告用户调整出入边
```
